# XR 3.1.6 Release Checklist

Release owner must tick each item before tagging/publishing.

## Clean checkout and dependencies

- [ ] Confirm branch and commit: `git rev-parse --abbrev-ref HEAD && git rev-parse HEAD`.
- [ ] Confirm clean tree except intended release artifacts: `git status --short`.
- [ ] Install with lockfile: `bun install --frozen-lockfile`.
- [ ] Confirm Bun version: `bun --version` (expected 1.3.14 for this release run).

## Static gates

- [ ] Version sync: `bun run set-version:check`.
- [ ] Typecheck: `bun run typecheck`.
- [ ] Inventory generation: `bun run baseline:inventory`.
- [ ] Package dry-run: `bun pm pack --dry-run`.

## Required tests

- [ ] Full test suite: `bun test`.
- [ ] Daemon/auth regression: `bun test test/daemon.test.ts`.
- [ ] Workspace/store regression: `bun test test/state/workspace-store.test.ts`.
- [ ] Security/control regressions: `bun test test/security.test.ts test/security/shield.test.ts test/control.test.ts`.
- [ ] Plugin/skill/MCP regressions: relevant `test/plugins*`, `test/skills*`, `test/ecosystem.test.ts`.

## Runtime smoke

- [ ] CLI version: `bun run src/index.ts --version`.
- [ ] CLI help: `bun run src/index.ts help`.
- [ ] Doctor JSON: `bun run src/index.ts doctor --json`.
- [ ] Doctor failure path verified with controlled fixture or pure summary tests.
- [ ] Workspace create/list/switch/reopen in isolated `XR_HOME`.
- [ ] Daemon health open and protected routes token-authenticated.
- [ ] Shutdown of daemon is clean.

## Installation/deployment

- [ ] `install.sh` validated in a clean supported Linux environment.
- [ ] `install.ps1` validated on a Windows runner, or Windows remains marked not tested.
- [ ] Docker build and localhost-bound start verified, or Docker remains optional/not tested.
- [ ] Data volume path and daemon port match docs (`7842` container, localhost host binding).

## Security/data safety

- [ ] Diagnostics redact secrets and only report secret presence.
- [ ] Daemon binds localhost and requires token except `/api/health`.
- [ ] Backup/restore procedure documented and manually rehearsed.
- [ ] No required tests need credentials or network.
- [ ] Optional integrations fail unavailable/closed, not as fake success.

## Release artifacts

- [ ] `docs/release/3.1.6/INVENTORY.md` and `inventory.json` current.
- [ ] `SUPPORT_MATRIX.md` current for the validation environment.
- [ ] `VALIDATION_REPORT.md` and `validation-report.json` generated by `bun run baseline:validate`.
- [ ] `BASELINE_MEASUREMENTS.md` and `baseline-measurements.json` generated by `bun run baseline:measure`.
- [ ] `RELEASE_NOTES.md`, `MIGRATION.md`, and rollback notes complete.
- [ ] `CHANGELOG.md` includes XR 3.1.6.
