# XR 3.1G — Website UX Audit

**Date:** 2026-07-14

## Current State Assessment

The existing XR website (pre-3.1G) is treated as **completely obsolete**.

### Issues Identified:
- Feels like a typical AI SaaS landing page
- Overuse of glassmorphism and neon
- No clear "Operating System" positioning
- Static cards instead of immersive experiences
- Missing 3D and motion systems
- Poor information hierarchy
- No real GitHub integration
- No immersive capability exploration

## Target UX Principles

1. Instant recognition as an AI Operating System
2. Every section communicates capability without walls of text
3. Immersive, interactive sections replace static marketing
4. Physical, high-quality motion
5. Zero visual clutter

## Competitive UX Gaps

- Claude & Linear: Better motion & calm
- Vercel & Apple: Better typography & whitespace
- v0-compute reference: Better 3D integration
- Raycast: Better speed and command-like feel

## Recommended UX Shifts

- From "product marketing" → "platform revelation"
- From "feature list" → "capability journey"
- From "static pages" → "interactive operating system preview"

## Success Metrics

- User immediately understands XR is an OS
- Time-to-understanding < 8 seconds
- High engagement with interactive sections
- Strong GitHub metrics visibility
