# XR 3.1G — Content Architecture

**Date:** 2026-07-14

## Content Principles

- Short, powerful statements
- Visual-first
- No walls of text
- Every capability shown, not just described

## Section Content Strategy

- Hero: One sentence + CTA
- Why XR Exists: 3–4 powerful statements
- Capabilities: Interactive first, text second
- All technical sections: Real demos + GitHub data

## Tone Guidelines

- "XR runs your entire machine."
- "One intelligence. Every interface."
- "Local. Cloud. Offline. Yours."

Content must be written with the final definition of done in mind: make competing sites feel incomplete through clarity and craftsmanship.
